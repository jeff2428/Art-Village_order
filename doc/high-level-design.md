# 藝素村線上點餐小程式 - 概要設計文檔

## 1. 文檔目的

本文根據 `doc/proposal.md` 的需求內容，定義藝素村線上點餐小程式的概要設計，包含系統模塊劃分、模塊職責、模塊關係、主要資料流與技術選型。

本設計採用指定技術架構：

- 顧客端前端：GitHub Pages 靜態網頁
- 顧客登入：LINE LIFF
- 後端 API：Google Apps Script Web App
- 管理後台：Google Apps Script HtmlService
- 資料儲存：Google Sheets
- 訂單推播：LINE Messaging API

## 2. 系統範圍

### 2.1 系統目標

系統提供顧客透過專屬網址與 LINE 登入後進行預約點餐，並將完整訂單明細即時推播至店家內部 LINE 群組。管理者可透過後台維護菜單、營業排程、特殊休假、完售狀態與公告訊息。

### 2.2 系統不包含

依需求文檔規範，第一版不包含以下功能：

- 線上金流付款
- 顧客自行修改訂單
- 顧客自行取消訂單
- 系統自動做人數上限控管或防超收

## 4. 模塊劃分

## 4.1 顧客端前端模塊

部署位置：GitHub Pages

顧客端負責讓顧客登入、瀏覽菜單、選擇餐點、填寫預約資訊、確認公告與送出訂單。

| 模塊 | 職責 | 主要輸入 | 主要輸出 |
| --- | --- | --- | --- |
| LIFF 登入模塊 | 初始化 LINE LIFF，取得顧客 LINE 身分 | LINE 授權資料 | LINE user id、顯示名稱 |
| 首頁公告模塊 | 顯示後台設定的首頁最上方公告 | 公告設定 | 首頁公告文字 |
| 彈窗公告模塊 | 顯示後台設定的彈出式公告 | 公告設定 | 彈窗公告 |
| 菜單分類模塊 | 顯示餐點分類，提供快速切換分類標籤 | 菜單分類資料 | 分類標籤 UI |
| 餐點列表模塊 | 顯示未完售餐點 | 餐點資料、完售狀態 | 可點選餐點列表 |
| 客製化選項模塊 | 以選項按鈕呈現麵種類、軟硬、鹹度、辣度等選項 | 餐點對應客製化設定 | radio / checkbox 選項 |
| 購物籃模塊 | 暫存顧客已選餐點與客製化內容 | 餐點選擇結果 | 訂單明細草稿 |
| 預約資訊模塊 | 收集訂位人姓名、電話、總人數、用餐日期與時間 | 顧客填寫資料 | 預約資訊 |
| 結帳前提醒模塊 | 在送出訂單前顯示後台設定的提醒文字 | 結帳前公告設定 | 顧客確認提醒 |
| 訂單送出模塊 | 將訂單資料送至 GAS Web App | 訂單明細、預約資訊、LINE 身分 | API 請求 |
| 完成畫面模塊 | 顯示點餐完成確認訊息 | API 回應 | 訂單完成畫面 |

## 4.2 後端 API 模塊

部署位置：Google Apps Script Web App

後端 API 負責提供菜單資料、接收訂單、執行規則驗證、寫入 Google Sheets，並觸發 LINE 推播。

| 模塊 | 職責 | 主要輸入 | 主要輸出 |
| --- | --- | --- | --- |
| API 路由模塊 | 根據請求類型分派到對應處理流程 | HTTP request | API response |
| 菜單查詢 API | 讀取分類、餐點、價格、客製化選項與完售狀態 | 前端查詢請求 | 菜單 JSON |
| 公告查詢 API | 讀取三個公告版位的文字與顯示開關 | 前端查詢請求 | 公告 JSON |
| 營業排程查詢 API | 讀取開放訂餐時間區間與特殊休假日期 | 前端查詢請求 | 可預約日期/時間設定 |
| 訂單驗證模塊 | 驗證訂單必填欄位、營業時間、休假日期、當日截止時間 | 訂單資料、系統設定 | 驗證結果 |
| 訂單處理模塊 | 整理訂單格式，產生訂單紀錄 | 驗證通過的訂單 | 訂單資料列 |
| 訂單寫入模塊 | 將訂單寫入 Google Sheets | 訂單資料列 | 寫入結果 |
| LINE 推播模塊 | 將完整訂單明細推播到店家內部 LINE 群組 | 訂單明細 | LINE 推播結果 |
| 錯誤回應模塊 | 將驗證失敗或系統錯誤轉為前端可理解訊息 | 錯誤內容 | JSON error response |

管理後台供店家管理系統設定與營運資料。

| 模塊 | 職責 | 主要輸入 | 主要輸出 |

| 餐點管理模塊 | 維護餐點名稱、內容、價格與分類 | 餐點資料 | 餐點設定 |
| 客製化選項管理模塊 | 維護餐點對應的客製化選項 | 選項群組、選項值 | 客製化設定 |
| 庫存/完售管理模塊 | 設定個別餐點是否完售 | 餐點完售狀態 | 前台隱藏依據 |
| 營業時間管理模塊 | 設定開放訂餐時間區間 | 時間區間設定 | 營業排程 |

## 4.4 資料儲存模塊

資料儲存位置：Google Sheets

Google Sheets 作為系統資料庫，保存菜單、訂單與設定資料。

| Sheet | 職責 | 主要欄位方向 |
| --- | --- | --- |
| Categories | 餐點分類 | 分類 ID、分類名稱、排序、啟用狀態 |
| Products | 餐點資料 | 餐點 ID、分類 ID、名稱、內容、價格、完售狀態、排序 |
| OptionGroups | 客製化選項群組 | 群組 ID、名稱、選擇類型、是否必填 |
| OptionItems | 客製化選項內容 | 選項 ID、群組 ID、選項名稱、排序 |
| ProductOptions | 餐點與客製化選項關聯 | 餐點 ID、群組 ID |

| Holidays | 特殊休假日期 | 日期、原因、是否啟用 |
| Announcements | 公告設定 | 版位、文字內容、顯示開關 |
| Orders | 訂單主檔 | 訂單 ID、LINE user id、姓名、電話、人數、日期、時間、狀態、建立時間 |
| OrderItems | 訂單品項明細 | 訂單 ID、餐點名稱、數量、價格、客製化內容 |

## 5. 模塊關係

## 5.1 顧客端與後端關係

```text
顧客端 LIFF 登入模塊
        |
        v
LINE Platform
        |
        v
顧客端取得 LINE profile

顧客端菜單分類模塊
顧客端餐點列表模塊
顧客端客製化選項模塊
顧客端公告模塊
        |
        | GET
        v
GAS 菜單查詢 API / 公告查詢 API / 營業排程查詢 API
        |
        v
Google Sheets
```

關係說明：

- 顧客端不直接讀取 Google Sheets。
- 顧客端透過 GAS API 取得菜單、公告與營業排程。
- LIFF 登入只負責取得顧客 LINE 身分，訂單仍由 GAS API 處理。

## 5.2 訂單模塊關係

```text
購物籃模塊
        |
        v
預約資訊模塊
        |
        v
結帳前提醒模塊
        |
        v
訂單送出模塊
        |
        | POST 訂單資料
        v
GAS API 路由模塊
        |
        v
訂單驗證模塊
        |
        v
訂單處理模塊
        |
        +------------------> 訂單寫入模塊 --> Google Sheets
        |
        +------------------> LINE 推播模塊 --> 店家 LINE 群組
        |
        v
前端完成畫面模塊
```

關係說明：

- 訂單送出前，顧客端必須顯示結帳前提醒。
- 訂單驗證由後端執行，避免只依賴前端判斷。
- 訂單寫入成功後，後端再觸發 LINE 推播。
- 顧客端只顯示完成結果，不提供自行修改或取消功能。

## 5.3 管理後台與資料關係

```text
管理後台
  |
  +--> 菜單分類管理模塊 --------+
  +--> 餐點管理模塊 ------------+
  +--> 客製化選項管理模塊 ------+--> Google Sheets
  +--> 庫存/完售管理模塊 -------+
  +--> 營業時間管理模塊 --------+
  +--> 特殊休假管理模塊 --------+
  +--> 公告管理模塊 ------------+
  +--> 訂單查看模塊 ------------+
```

關係說明：

- 管理後台所有設定最終寫入 Google Sheets。
- 顧客端取得的菜單與公告資料都來自同一份 Google Sheets 設定。
- 餐點設定為完售後，前端菜單查詢結果不應包含該餐點。

## 6. 主要資料流

## 6.1 顧客點餐資料流

1. 顧客開啟專屬網址，載入 GitHub Pages 前端。
2. 前端初始化 LINE LIFF。
3. 顧客完成 LINE 登入授權。
4. 前端取得顧客 LINE profile。
5. 前端呼叫 GAS API 取得菜單、公告、營業時間與休假設定。
6. 前端顯示首頁公告與菜單分類標籤。
7. 顧客選擇餐點與客製化選項。
8. 顧客填寫姓名、電話、人數、用餐日期與時間。
9. 前端顯示送出訂單前公告。
10. 顧客確認後送出訂單。
11. GAS 後端驗證訂單資料與營業規則。
12. GAS 後端寫入 Google Sheets 訂單資料。
13. GAS 後端呼叫 LINE Messaging API 推播訂單到店家內部 LINE 群組。
14. 前端顯示訂單完成確認訊息。

## 6.2 管理設定資料流

1. 管理者開啟 GAS HtmlService 後台。
2. 管理者維護菜單、分類、客製化選項、完售狀態、營業時間、休假日期或公告。
3. 後台將設定寫入 Google Sheets。
4. 顧客端下一次查詢 GAS API 時取得最新設定。
5. 完售餐點在前台自動隱藏。
6. 公告依設定顯示於首頁、彈窗或送出訂單前。

## 7. 核心規則設計

## 7.1 付款規則

- 系統不串接線上金流。
- 訂單僅記錄點餐與預約資訊。
- 結帳方式固定為現場結帳。

## 7.2 點餐時限規則

- 當日訂單必須在營業結束前 1 小時以前送出。
- 此規則由 GAS 後端訂單驗證模塊執行。
- 前端可提前提示，但不可作為唯一驗證來源。

## 7.3 預約天數規則

- 系統不限制未來可預訂天數。
- 只要日期未被特殊休假設定阻擋，且符合營業時間規則，即可送出。

## 7.4 人數控管規則

- 系統不設定人數上限。
- 系統不自動防超收。
- 滿座或需調整時，由店家人工聯絡顧客協調。

## 7.5 菜單顯示規則

- 前台提供分類標籤快速切換。
- 完售餐點不顯示於前台。
- 客製化需求採選項按鈕，不以自由文字作為主要輸入方式。

## 7.6 訂單修改與取消規則

- 顧客送出訂單後，不可自行修改或取消。
- 顧客需透過電話或官方 LINE 聯繫人工處理。
- 此規則應透過送出訂單前公告提醒顧客。

## 8. 技術選型

| 區域 | 技術 | 說明 |
| --- | --- | --- |
| 顧客端前端 | GitHub Pages | 託管靜態 HTML、CSS、JavaScript |
| LINE 登入 | LINE LIFF | 取得顧客 LINE 身分，降低註冊成本 |
| 後端 API | Google Apps Script Web App | 提供菜單查詢、公告查詢、訂單送出等 API |
| 管理後台 | Google Apps Script HtmlService | 建立管理者操作介面 |
| 資料儲存 | Google Sheets | 儲存菜單、設定、公告與訂單紀錄 |
| 店家通知 | LINE Messaging API | 將訂單明細推播到內部 LINE 群組 |

## 9. 外部介面

## 9.1 顧客端呼叫 GAS API

| API | 用途 | 回應資料 |
| --- | --- | --- |
| `GET menu` | 取得菜單分類、餐點與客製化選項 | 菜單 JSON |
| `GET announcements` | 取得三個公告版位設定 | 公告 JSON |
| `GET schedule` | 取得營業時間與特殊休假設定 | 排程 JSON |
| `POST order` | 送出訂單 | 成功/失敗結果 |

## 9.2 GAS 呼叫 LINE Messaging API

用途：訂單成立後，將完整訂單資訊推播至店家內部 LINE 群組。

推播內容包含：

- 訂位人姓名
- 聯絡電話
- 總人數
- 用餐日期與時間
- 餐點品項
- 客製化細節

## 10. 風險與注意事項

| 風險 | 影響 | 設計處理 |
| --- | --- | --- |
| 前端時間判斷被繞過 | 可能送出不符合規則的訂單 | 後端必須再次驗證點餐時限與休假設定 |
| Google Sheets 欄位被人工改壞 | API 讀取或寫入失敗 | 後端需對必要欄位做存在性檢查與錯誤回應 |
| LINE 推播失敗 | 店家未即時收到訂單 | 訂單仍需先寫入 Sheets，並回傳推播失敗狀態供追蹤 |
| 完售狀態未即時反映 | 顧客可能看到不可點餐品 | 菜單查詢 API 每次依最新 Sheets 狀態過濾 |
| 顧客誤以為可自行取消 | 現場處理成本增加 | 在送出訂單前公告明確提醒修改/取消規則 |
