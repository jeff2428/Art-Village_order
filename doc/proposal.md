# 星空x藝素村線上點餐小程式 - 需求文檔

## 目標
開發一套專屬線上點餐小程式，顧客透過專屬網址並使用 LINE 登入後，即可進行預約點餐。系統旨在簡化顧客點餐流程、精準傳達客製化需求，並將訂單資訊即時推播至內部 LINE 群組，提升現場營運與廚房備料效率。

## 輸入 (Input)

### 顧客端輸入：
- **登入授權**：LINE 帳號登入綁定
- **預約資訊**：訂位人姓名、聯絡電話、總人數、用餐日期與時間
- **餐點選擇**：依分類點選餐點項目
- **客製化選項**：透過選項按鈕勾選（如：麵種類、麵軟硬、鹹度、辣度等）

### 管理端（後台）輸入：
- **營業與排程**：開放訂餐時間區間、特殊休假日期設定
- **菜單管理**：餐點分類建立、餐點內容、價格、對應的客製化選項
- **庫存狀態**：設定個別餐點是否完售（完售即觸發前台隱藏）
- **公告訊息設定**：三個獨立版位的文字內容與顯示開關（首頁最上方、彈出式視窗、送出訂單前）

## 輸出 (Output)

### 對內推播（給店家）：
將顧客填妥的完整訂單明細（包含預約人資訊、用餐時間、餐點品項及客製化細節），自動發送至專用的 LINE 群組

### 對外顯示（給顧客）：
- 點餐完成的確認訊息或畫面
- 依據後台設定，顯示首頁公告、彈窗公告或結帳前提醒
- 隱藏已完售的餐點品項，維持版面簡潔

## 規範 (Rules)

1. **付款規範**：不串接線上金流，一律採「現場結帳」
2. **點餐時限**：系統將於當日「營業結束前 1 小時」自動停止接受當日的線上點單
3. **預約天數**：開放無限期提前預訂，不限制未來天數
4. **人數控管**：系統端「不」設定人數上限或防超收機制，一律接單；遇滿座狀況由人工介入聯絡協調
5. **菜單介面**：需具備「快速切換分類」的標籤功能，且客製化需求須採用「選項按鈕」設計，避免自由填寫造成廚房視讀落差
6. **訂單修改與取消**：訂單一旦送出，系統不開放顧客自行修改或取消；須強制規定顧客透過電話或官方 LINE 聯繫人工處理（建議可設定於「送出訂單前」的公告訊息中提醒顧客）
